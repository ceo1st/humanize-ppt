# Humanize PPT V0.3 Preview-First Edition

## 结论

V0.3 是一个小版本号里的能力跃迁：它不再只是把 `beautiful-html-templates` 写进路由计划，而是能真实读取模板库、选择 3 个候选模板，并生成可打开的标题页预览 gallery。

因此这一步应算 **V0.3**，不是 V0.2 patch。

原因：

1. V0.2 的边界是 Router Edition：生成 AST 契约、路由计划、commands、manifest、QA；
2. V0.3 新增真实 renderer adapter：`beautiful-html-templates` 不再只是 planned command，而是会产出 HTML artifact；
3. 用户体验从“告诉你该走哪条路”变成“先给你 3 个风格方向看，再决定完整 deck”。

## 一条命令

```bash
python3 scripts/humanize_ppt_v3.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-v0.3-preview \
  --title "AI 工具更新，不只是功能清单" \
  --style-mode preview-first
```

显式指定模板库：

```bash
python3 scripts/humanize_ppt_v3.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-v0.3-preview \
  --title "AI 工具更新，不只是功能清单" \
  --style-mode preview-first \
  --beautiful-repo /path/to/beautiful-html-templates
```

可选风格提示：

```bash
--occasion "AI workflow product demo, developer tools, creator presentation" \
--mood "confident editorial modern design-led practical"
```

## 当前已接通

- 读取 `beautiful-html-templates/index.json`；
- 按 title/source/occasion/mood 对 `mood`、`tone`、`occasion`、`best_for` 打分；
- 选择 3 个互相有区分度的模板；
- 复制每个模板需要的 sibling assets 和 `runtime/deck-stage.js`；
- 从模板中取第一张 cover slide；
- 替换为用户真实 title、subtitle、kicker；
- 只保留第一张 slide，生成 title-slide preview；
- 生成 `outputs/beautiful/previews/index.html` gallery；
- 生成 `outputs/beautiful/preview_manifest.json`；
- 更新 `router_plan.json` route status 为 `rendered`；
- QA 会把 preview render failure 作为 blocking issue。

## V0.4 接续

V0.3 负责“先看见风格”：`--style-mode preview-first` 生成 3 个真实标题页候选。

V0.4 已接上下一步：`--selected-template <slug>` 会用选中的 Beautiful 模板生成完整 deck，输出到 `outputs/beautiful/selected/index.html`，并写入 `selected_manifest.json`。
