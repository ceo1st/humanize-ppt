# Humanize PPT V0.4 Selected Template Full Deck

## 结论

V0.4 把 V0.3 的 preview-first 路线补上了下一步：用户从 `beautiful-html-templates` 的 3 个候选标题页里选中一个模板后，可以用 `--selected-template <slug>` 直接生成完整 HTML deck。

这一步仍然保持 Humanize PPT 的边界：Humanize PPT 不变成一个大而全的模板库，而是把 AST 契约、模板选择、全量 deck 产物和 QA 状态串起来。

## 一条命令

```bash
python3 scripts/humanize_ppt_v4.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-v0.4-selected \
  --title "AI 工具更新，不只是功能清单" \
  --selected-template <slug>
```

显式指定模板库：

```bash
python3 scripts/humanize_ppt_v4.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-v0.4-selected \
  --title "AI 工具更新，不只是功能清单" \
  --selected-template <slug> \
  --beautiful-repo /path/to/beautiful-html-templates
```

## 当前已接通

- `--selected-template <slug>` CLI 参数；
- 自动把 selected-template route 归到 `beautiful-html-templates`；
- 校验 `beautiful-html-templates/index.json` 和 `templates/<slug>/template.html`；
- 复制模板 sibling assets 与 `runtime/deck-stage.js`；
- 用 `slide_plan.json` 生成全量 slide sections；
- 替换模板内原有 `<deck-stage>` / `#deck` / `<body>` 内容；
- 生成 `outputs/beautiful/selected/index.html`；
- 生成 `outputs/beautiful/selected_manifest.json`；
- 更新 `router_plan.json` 的 selected template、actual output 和 status；
- QA 会把 missing library / missing template 作为 blocking issue。

## 当前边界

- V0.4 的 Beautiful full deck 是确定性 adapter，不是设计师级二次排版；
- 现在优先保证“选中模板能生成完整 deck”这条链路真实可跑；
- 还没有接 presenter mode；
- 还没有自动 deploy / PDF export；
- 如果模板本身的布局结构很特殊，V0.4 会保留模板资源与外壳，但 slide 内容采用 Humanize PPT 的标准 section 注入方式。
