# V0.5 Presenter / Export Adapter

## Goal

V0.5 turns Humanize PPT from “can generate or route a final deck” into “can complete a usable presentation package”. It keeps Humanize PPT as the AST director and post-processing adapter layer, not as a fixed visual template library.

## New CLI

```bash
python3 scripts/humanize_ppt_v5.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-v0.5-complete \
  --title "AI 工具更新，不只是功能清单" \
  --selected-template <slug> \
  --presenter-adapter \
  --export-adapter
```

The implementation remains in `scripts/humanize_ppt_v2.py` for backward compatibility; `scripts/humanize_ppt_v5.py` is the stable public V0.5 wrapper.

## Presenter Adapter

Triggered by `--presenter-adapter` after a final deck exists.

Outputs:

- `outputs/presenter/index.html`
- `outputs/presenter/presenter_manifest.json`
- `outputs/presenter/render_report.md`

Behavior:

- Embeds the final deck in an iframe.
- Displays `CURRENT`, `NEXT`, and `SCRIPT` panels.
- Builds notes from `slide_plan.json` and speaker intent fields.
- Sends a lightweight `postMessage({type:'preview-goto', idx})` signal to the deck iframe for future renderer integrations.

Boundary:

- It is a shell over an existing final deck.
- It does not replace renderer-native presenter mode from `html-ppt`.
- It does not run when only a Beautiful preview gallery exists.

## Export Adapter

Triggered by `--export-adapter` after a final deck exists.

Outputs:

- `outputs/export/package/index.html`
- `outputs/export/export_pdf.sh`
- `outputs/export/export_manifest.json`
- `outputs/export/README.md`
- `outputs/export/render_report.md`

Behavior:

- Copies the final deck directory into `outputs/export/package/`.
- Creates a portable HTML entrypoint at `outputs/export/package/index.html`.
- Writes a Playwright-based `export_pdf.sh` helper for local PDF export.
- Records source deck path, package path, script path, and slide count in `export_manifest.json`.

Boundary:

- PDF export requires Playwright Chromium on the machine running the script.
- The adapter packages static deck files; it does not guarantee every renderer animation maps perfectly into PDF pages.

## QA rule

Presenter/export adapters are only complete when `router_plan.json` marks:

- `presenter-adapter.status == rendered` if requested.
- `export-adapter.status == packaged` if requested.
- `outputs/qa/qa_report.md` is `status: pass`.

If the route only produced `outputs/beautiful/previews/index.html`, presenter/export must return `missing-deck` and QA must fail, because there is no final deck yet.
