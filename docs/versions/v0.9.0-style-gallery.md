# v0.9.0 — 风格画廊（Style Gallery）+ 视觉增强接线

Date: 2026-06-17 · 基线：v0.8.0（演讲体检）

## 这一版的一句话

为「为演讲而生 + 广义兼容下游精美 HTML PPT」这个定位补上**渲染之前的封面选择门**：出大纲之前，先让人在 ≥4 张真渲封面里挑风格。封面由下游真渲，Humanize 只出 spec/command，自己不渲一个像素——和 v0.6.4 起的边界完全一致。

## 改了什么

### 1. media schema 机器可执行化（commit 1，port 自 v0.6.7-hero-showcase）

`contracts/slide-plan.schema.json` 的 `media.{image,diagram,video}` 各加 `asset_path` + `prompt_hint`（image 另加 `aspect_ratio` / `max_size_kb`）。`decide_media` 加 `slide_id` 参数 + `media_extension` / `media_prompt_hint` helper。一个带 `asset_path` 的素材位是可执行任务，不带的只是标签（补上 v0.6.5 的缺口）。

### 2. 三 writer 共享 media 段 + 视觉增强生产指引（commit 2）

三个 brief writer（guizang / frontend-slides / beautiful）共享 `_format_per_page_media_block`（surface 每页素材位）+ 新增 `_media_production_guidance`（把每个 `kind` 映射到具体的热插拔生成 skill：图片→imagegen/imagen/nanobanana/codex image；视频→remotion 系列；diagram→确定性内联 SVG）。frontend-slides 和 beautiful 原是骨架，现都带 media 段。

### 3. 头号特性：首页风格画廊门（commit 3）

`--style-gallery` 在出大纲前停下，`main()` 中排在 `--preview-outline` 之前（先选风格再审大纲）。

- 模块级 `STYLE_GALLERY_CANDIDATES`：guizang 跨 Style A 三主题（ink-classic / kraft-paper / indigo-porcelain）+ Style B 瑞士 ikb，四张视觉互异；frontend-slides 四风格方向；beautiful 四模板槽。每个候选都 ≥4，`--gallery-count`（默认 4、最小 4）可加、上限取候选数。
- 每候选写 `commands/style-gallery/<id>.md`：只渲封面一页（S01）→ `outputs/style-gallery/<id>/cover.{html,png}`，不渲整套。
- 零依赖单文件 `style_gallery.html`：iframe 缩放缝合 4 张封面供挑；未渲染的封面位显示 always-visible 诚实 caption，不摆拍假缩略（和 showcase 同一条班规）。
- `style_gallery_plan.json`：每候选的 id / label / cli / command_file / cover 路径 / 回灌命令。挑中后跑回灌命令把风格带进正常 outline → brief 流程。
- 复用 `read_source` / `detect_language` / `build_slide_plan` / `choose_routes` / `now_iso`，无新解析路径。

浏览器实测（playwright + http.server）：iframe 缩放真渲封面正常，空白态诚实。规格见 `references/style-gallery-spec.md`。

### 4. 新失败类：WebGL hero 封面静态截图陷阱（#5）

`references/qa-failure-modes.md` 的「静态扫描还测不出的失败类」补一条：Style A 封面用 WebGL hero canvas，HTML 全过静态检查但静态 PNG 截图捕获到空白（canvas 加载后才异步画首帧，截图早于绘制）。这是「正确的页 + 错误的截图 = 空白产物」，和文字溢出那种页本身有问题不同。实证：9 风格试验里 Style A ink-classic 封面截图仅 14KB 是空白页（同批 Style B 瑞士静态正常），该批撤回不入库。按目录纪律不包装成 `FAILURE_MODES`（Humanize 不读 PNG 字节）。兜底规则写进了风格画廊的封面命令：以活页 cover.html 为准、截图延迟 ≥1.5s、<20KB 判截图失败。

### 5. 文档与回炉

- `references/qa-failure-modes.en.md`：失败模式目录英文镜像，逐条对齐同一份 `FAILURE_MODES` id，含新 WebGL 失败类。
- `SPEC.md`：补上全项目缺的引擎技术规格（定位边界、CLI 全表、数据流、输出契约、风格画廊、演讲体检、媒体模型、注册表）。
- README 中英同步加「风格画廊」小节 + GIF 槽位（诚实留位，真录制留到下游真渲 4 张封面之后）；`scripts/record_demo_gif.py` 落地（录两张零依赖工作底稿为 GIF，依赖诚实预检）。
- 版本号 0.8.0 → 0.9.0：引擎 VERSION、运行期 banner、preview 脚本、SKILL.md、registry、marketplace、版本元数据测试同步。
