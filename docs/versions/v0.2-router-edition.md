# Humanize PPT V0.2 Router Edition

## 结论

V0.2 的目标是让 Humanize PPT 可以被单独调用，但不把它变成一个固定视觉模板库。

它负责：

1. 把原始资料变成 AST 生产契约；
2. 根据场景选择下游 renderer；
3. 给下游 Agent 写有边界的 `commands/*.md`；
4. 记录 `router_plan.json` 和 `run_manifest.json`；
5. 先跑一个可验证的 QA gate。

## 一条命令

```bash
python3 scripts/humanize_ppt_v2.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-v0.2 \
  --title "AI 工具更新，不只是功能清单" \
  --renderer auto
```

## 预览优先

```bash
python3 scripts/humanize_ppt_v2.py \
  --source examples/01-ai-tool-update/source.md \
  --out .humanize-ppt-runs/ai-tool-update-preview \
  --title "AI 工具更新，不只是功能清单" \
  --style-mode preview-first
```

## 当前已接通

- `registry/renderer_registry.json`：记录 guizang、beautiful-html-templates、html-ppt、frontend-slides、qa 的能力。
- `scripts/humanize_ppt_v2.py`：生成 AST 契约、路由计划、命令文件、manifest、QA 报告。
- `guizang` 稳定路径：当 primary renderer 是 `guizang` 时，会用本地 guizang `assets/template.html` 生成 `outputs/guizang/index.html`。
- `qa` 路径：生成 `outputs/qa/qa_report.md`、`fix_list.md`、`final_manifest.json`。

## V0.3 后续说明

V0.3 已经接通 `beautiful-html-templates` 的真实 preview-first 路径：会生成 3 个标题页 HTML preview、gallery、`preview_manifest.json` 和 render report。

V0.2 本文档保留为 Router Edition 的历史说明。

## Product framing

Humanize PPT V0.2 是 PPT Agent Team 的主控层，不是 renderer 替代品。

一句话：

> 先用 AST 把资料变成人感大纲，再把大纲路由给最合适的 PPT Skill。
